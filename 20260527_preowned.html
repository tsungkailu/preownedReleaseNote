RELEASE NOTICE

中古車上版通知

PRE1.138.0  |  2026-05-27
本次版本共包含 8 項更新，涵蓋新功能開發、流程自動化及介面修正。
新功能

01　AI 合成中古車圖

中古車後台缺乏自動化車圖合成能力，人工處理耗時且難以管理。導入 AI 合成車圖功能，後台可管理車圖、查詢合成狀態，並防止重複處理；合成呼叫改以 Service Account 取代 API key，提升安全性；每張圖生成費用約3元，請愛惜節省珍惜使用。

02　中古車出售後與 CRS 資產狀態同步

中古車開立或作廢發票時，CRS 未能即時同步更新資產狀態，造成資訊不一致。開立發票後自動通知 CRS 將資產狀態更新為「出售」；發票作廢時同步改回「未出售」；車籍新增「資產狀態」欄位，並提供內部更新 API。

03　中古車業代申請開立發票

業代缺乏在系統內直接申請開立發票的流程，需依賴人工處理。新增業代申請開立發票功能，並於申請時提醒業代檢核重要資訊。增加牌照繳銷及車證寄送地點。

04　中古車交車功能

交車時，可透過神馬維運同時完成出庫及客戶簽收，延續實車出入庫功能，車輛出庫時可選擇「出售交車」，並新增上傳客戶簽名及行照功能。

05　中古車調度與成交狀態自動化

調度狀態改為開放時，若存在當前有效成交資料，系統自動將狀態轉為「成交」，避免成交車輛被誤用的情況。

06　中古車核價失效後隱藏底價

核價失效後底價仍顯示於車籍頁面，可能造成業務誤判。核價失效後，車籍頁面不再顯示底價，避免過期定價資訊造成業務誤操作。

功能修正與優化

07　中古車後台車籍管理介面修正

車籍管理的營業所欄位更新時，未更新訂單明細。前端車籍管理頁面改從 API 回傳的 stationId 取值，而非 stationInfo，確保營業所資料正確顯示。

08　訂單業代資訊同步更新

訂單列表的業代單位及名稱，與明細不同。前端更新訂單時，若業代單位或名稱有異動，一併回傳後端更新，確保訂單資料完整性。
